## AI-VMS 개발

### Tapo 카메라 ONVIF/RTSP 연동 이슈 해결

- **ONVIF WS-Security 인증 추가**: Tapo C500/C200은 HTTP Digest Auth가 아닌 WS-Security UsernameToken 방식 사용. `client.py`에 WS-Security → Digest Auth 순으로 폴백하는 인증 로직 구현
- **RTSP TCP 전송 제조사별 분기**: Tapo의 RTSP 서버는 동일 연결 재인증 거부 → go2rtc `#tcp` 플래그 필요. `_TCP_TRANSPORT_MANUFACTURERS` 셋으로 제조사별 TCP 전송 적용 (Hikvision은 UDP 유지)
- **환경설정 경로 수정**: `settings.py`의 `env_file` 경로를 `("../.env", ".env")`로 변경, `extra: "ignore"` 추가
- **go2rtc 스트림 정리**: 10개 이상의 테스트 스트림이 C500 연결 제한(~2세션)을 초과 → `go2rtc.yaml`에서 활성 카메라 3대만 남기고 정리
- 이슈 문서 작성: `docs/issue-tapo-onvif-rtsp-streaming.md`
